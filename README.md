## 复制英文，屏幕上直接出现中文

悬浮翻译器是一款轻量、低打扰的桌面翻译工具。它会监听你复制的英文内容，调用你配置的模型翻译成简体中文，并把结果显示在屏幕下方的透明字幕窗口中。

你不需要切换窗口、打开翻译网站，也不需要反复粘贴内容。复制、选中、按下快捷键，翻译结果就会出现在当前工作流里。

## 核心功能

### 复制即翻译

复制英文句子、段落或技术文档，应用会自动完成翻译。内置防抖和去重机制，不会因为一次复制触发多次请求，也不会重复显示相同结果。

### 选区翻译

在任意支持选中的应用中选中文本，按下快捷键即可翻译当前选区：

| 平台 | 默认快捷键 |
| --- | --- |
| Windows | `Ctrl+Alt+T` |
| macOS | `Command+Option+T`（⌘⌥T） |

在设置页点击快捷键框后，直接按下想要的组合键即可录入，不需要手动输入文字。应用会监听并识别 Ctrl/Command、Alt/Option、Shift 加字母、数字或 F1-F24 的组合。

### 不打扰工作的字幕窗口

- 透明、无边框、置顶显示；
- 不抢输入焦点，不影响当前应用操作；
- 支持调整字幕宽度、位置、字体、字号、行数和透明度；
- 支持淡入、停留和淡出动画；
- 新翻译会自然覆盖旧字幕，迟到的旧结果不会把新结果顶掉。

### 自动过滤无关内容

应用会在发送请求前过滤：

- 空内容和重复内容；
- URL 和明显的代码片段；
- 超长文本；
- 疑似 API Key、Token、密码和私钥的敏感内容；
- 不符合“仅翻译英文”条件的文本。

“仅翻译英文”会根据 ASCII 英文字母比例和常见英文词汇判断，并拒绝带重音的其他拉丁文字，避免把法语、西班牙语等内容误当成英文。

### 支持常见模型服务

支持 OpenAI-compatible 接口。你可以使用 OpenAI 或其他兼容相同 API 格式的模型服务，只需要填写 Base URL、模型名称和 API Key。

设置页提供完整的图形化配置，保存后立即生效，不需要重启应用。

## 安装与第一次使用

### Windows

1. 下载并解压 Windows x64 ZIP，或直接运行 `floating-translator.exe`。
2. 如果系统尚未安装 WebView2，请先安装 [Microsoft Edge WebView2 Evergreen Runtime](https://developer.microsoft.com/microsoft-edge/webview2/)。
3. 从系统托盘打开“设置…”。
4. 填写模型名称、Base URL 和 API Key，点击保存。
5. 复制一段英文，等待屏幕下方出现翻译字幕。

### macOS

1. 下载 DMG，打开后将“悬浮翻译器”拖入 `Applications`。
2. 第一次启动时允许“辅助功能”权限。
3. 如果系统没有自动打开权限设置，进入“系统设置 → 隐私与安全性 → 辅助功能”，手动允许悬浮翻译器。
4. 从菜单栏图标打开“设置…”，填写模型信息并保存。
5. 复制英文，或选中文本后按 `⌘⌥T`。

macOS 的辅助功能权限用于读取选区、注册全局快捷键和兼容复制；普通剪贴板监听不需要该权限。

当前 macOS 包使用 ad-hoc 签名，未配置 Apple Developer notarization。如果系统提示无法验证开发者，可在 Finder 中右键应用选择“打开”，或在“隐私与安全性”中允许本次启动。

## 配置模型

最少需要配置以下内容：

```yaml
llm:
  provider: "openai_compatible"
  base_url: "https://api.openai.com/v1"
  api_key: "${LLM_API_KEY}"
  model: "your-model-name"
```

完整示例见 [config.example.yaml](config.example.yaml)。配置文件位置为：

- Windows：`%APPDATA%\FloatingTranslator\config.yaml`
- macOS：`~/Library/Application Support/FloatingTranslator/config.yaml`

也可以通过 `LLM_API_KEY` 和 `LLM_BASE_URL` 环境变量提供或覆盖 API Key 和 Base URL。设置页不会显示已有 API Key 明文：

- 未修改 API Key 并保存：保留原 Key；
- 输入新 Key 并保存：替换原 Key；
- 输入后清空并保存：删除原 Key，应用会提示重新配置。

## 选区兼容模式

默认情况下，应用优先使用系统原生能力直接读取选区。某些应用不提供可访问的选中文本时，可以在设置页打开“强制兼容”。

兼容模式会临时模拟 `Ctrl+C` 或 `Command+C`，但会尽量保护原剪贴板：

- 原剪贴板包含复杂格式时直接取消，不冒险覆盖；
- 会保存并恢复所有允许的纯文本表示；
- 复制期间如果检测到新的剪贴板内容，会取消恢复并保留最新内容；
- 选区读取期间到达的剪贴板文本不会被静默丢弃。

## 隐私与安全

- 不保存剪贴板历史；
- 默认不记录完整剪贴板原文，只记录长度、哈希、过滤原因、模型名和耗时；
- 疑似密钥、Token、密码和私钥的内容默认不会发送给模型；
- 外部服务返回的错误会脱敏并截断，避免把 URL 查询参数或认证信息写入日志；
- 只有进入翻译流程的文本才会发送到你配置的模型服务；
- 开启“记录原始文本”后，日志可能包含普通剪贴板文本，建议仅在排查问题时临时使用；
- 应用不会把 API Key 回显到设置页。

## 常见问题

### 复制后没有翻译

请确认：

1. 模型名称、Base URL 和 API Key 已配置；
2. 托盘或菜单栏状态不是“配置错误”或“已暂停”；
3. 文本不是 URL、代码、敏感内容或超长内容；
4. “仅翻译英文”没有把当前内容过滤掉；
5. 模型服务可以从当前网络访问。

修改配置后，在托盘或菜单栏中选择“重新加载配置”，或从设置页保存。

### macOS 快捷键没有反应

请到“系统设置 → 隐私与安全性 → 辅助功能”确认悬浮翻译器已获授权。修改权限后，退出并重新打开应用通常即可恢复。

### macOS 能监听复制，但不能读取选区

这是正常的权限差异：剪贴板监听不需要辅助功能权限，选区读取和全局快捷键需要辅助功能权限。请授权后再尝试；如果目标应用仍不支持直接读取，可以开启“强制兼容”。

### 如何修改快捷键

打开设置页的“划词翻译”，点击“全局快捷键”输入框，然后按下组合键，例如 `Ctrl+Alt+T` 或 `⌘⌥T`。保存后，系统原生全局快捷键监听会立即使用新的组合键。

### 如何暂停自动翻译

从 Windows 托盘或 macOS 菜单栏中选择“暂停监听”。选区翻译可以单独启用或关闭，重新在设置页按下组合键即可修改快捷键。

## 当前范围

当前版本专注于“英文到简体中文”的桌面字幕翻译：

- 支持主屏幕字幕，不跟随活动窗口或指定屏幕；
- macOS 仅支持 Apple Silicon arm64；
- 暂不包含翻译历史、OCR、自动更新、安装器和专用 Ollama 适配器；
- 应用不内置翻译模型，需要连接一个可用的 OpenAI-compatible 模型服务。
