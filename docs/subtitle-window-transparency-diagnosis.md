# 字幕窗口透明问题诊断与后续改造建议

## 文档状态

- 记录日期：2026-07-10
- 当前结论：暂停继续修补 Wails 字幕窗口，后续优先改为独立原生透明字幕窗口
- 本文只记录诊断结论和建议方案，不代表原生字幕窗口已经实现

## 问题现象

期望效果是屏幕底部只显示字幕卡片，字幕以外的区域完全透明，并且不显示标题栏、边框、任务栏入口，也不拦截鼠标操作。

当前实际效果是：

- 字幕窗口的宽度、位置和距底部参数已经生效；
- 字幕文本及其半透明深色背景能够正常渲染；
- 字幕外围仍存在一个较大的白色矩形窗口；
- 窗口仍带有 Windows 标题栏、边框和关闭按钮；
- 因此整个窗口会遮挡底层应用，不能作为可用的悬浮字幕层。

这说明问题不在字幕尺寸计算，而在 Wails/WebView2 原生宿主窗口的透明与无边框行为。

## 当前实现链路

当前字幕仍是一扇独立的 Wails WebView 窗口：

1. ` main.go:38 ` 创建字幕 WebView 窗口；
2. ` main.go:76 ` 定义字幕窗口参数，包括无边框、透明背景、鼠标穿透和 Composition Hosting；
3. ` app.go:184 ` 等待 Vue 字幕页面完成事件订阅；
4. ` app.go:493 ` 根据屏幕工作区和配置计算字幕窗口尺寸及位置；
5. ` app.go:525 ` 将翻译结果作为 Wails 事件发送给字幕 WebView；
6. ` frontend/src/App.vue:73 ` 在 Vue 中控制字幕显示和隐藏；
7. ` frontend/src/style.css:15 ` 已明确将 `html`、`body` 和 `#app` 背景设置为透明；
8. ` frontend/src/style.css:55 ` 负责渲染真正需要显示的字幕卡片。

当前依赖的 Wails 版本为 `v3.0.0-alpha2.117`。

## 已完成的验证

### 1. 验证布局计算

通过原生窗口矩形检查确认，白色大窗口的宽度、高度和位置与字幕配置计算结果一致。

结论：`width_percent`、`bottom_offset_percent` 和窗口布局逻辑不是本次白底问题的原因。

### 2. 验证前端透明样式

前端已经对页面根节点设置透明背景，字幕卡片以外没有主动绘制白色背景。

结论：白色背景不是普通 CSS `background` 遗漏导致的。

### 3. 启用 WebView2 Composition Hosting

尝试为字幕窗口启用：

- `BackgroundTypeTransparent`；
- RGBA 全透明背景；
- `Frameless`；
- `IgnoreMouseEvents`；
- `WebView2CompositionHosting`。

对应的实验代码当前位于 ` main.go:76 `，回归配置测试位于 ` main_windows_test.go:11 `。

原生诊断日志确认 WebView2 Composition Controller 成功创建，没有发生回退到普通 HWND Controller。然而实际截图仍然存在白色背景、标题栏和边框。

结论：仅启用 Composition Hosting 无法解决当前环境中的问题。

### 4. 运行时强制修改原生窗口样式

在不修改项目代码的情况下，运行时将字幕窗口临时改为：

- `WS_POPUP`；
- 移除 `WS_EX_LAYERED`；
- 添加 `WS_EX_NOREDIRECTIONBITMAP`；
- 保留置顶、工具窗口和鼠标穿透相关样式；
- 强制触发窗口框架重新计算。

模型翻译成功且字幕事件已经发出，但人工观察结果仍是白底或标题栏存在。

结论：继续组合 Wails 窗口参数和 Win32 样式，无法得到可靠的透明字幕窗口。

## 诊断结论

在当前 Windows 环境、WebView2 Runtime 和 `Wails v3.0.0-alpha2.117` 的组合下，Wails WebView 窗口不能可靠满足以下四项同时成立：

- WebView 内容可见；
- WebView 外围完全透明；
- 原生窗口无标题栏和边框；
- 整个窗口鼠标穿透。

该结论只针对当前项目和已验证的运行环境，不表示所有 Wails 版本或所有 Windows 环境都一定存在相同行为。

继续修补 Wails 字幕窗口的主要问题是：

- 参数级单元测试只能证明配置值已设置，不能证明 Windows 最终合成效果正确；
- 不同 WebView2 Runtime、显卡驱动、DWM 状态和远程桌面环境可能产生不同结果；
- 即使某个样式组合暂时可用，后续升级 Wails 或 WebView2 后仍可能再次出现白底；
- 字幕窗口本身不需要完整 WebView，继续使用 WebView 增加了不必要的渲染和生命周期复杂度。

## 推荐改造方案

### 总体方案

设置窗口继续使用 Wails，字幕窗口改为独立的 Windows 原生分层窗口。

建议使用平台接口隔离实现：

```text
翻译处理器
    -> 字幕控制器接口
        -> Windows 原生分层窗口实现
        -> 未来其他平台实现

设置页面
    -> Wails 设置窗口
```

Windows 字幕窗口使用以下原生能力：

- `WS_POPUP`：不创建标题栏和标准窗口边框；
- `WS_EX_LAYERED`：支持逐像素 Alpha；
- `WS_EX_TRANSPARENT`：鼠标穿透；
- `WS_EX_TOOLWINDOW`：不显示任务栏入口；
- `WS_EX_TOPMOST`：保持字幕置顶；
- `WS_EX_NOACTIVATE`：显示字幕时不抢夺当前应用焦点；
- `UpdateLayeredWindow`：一次性提交带透明通道的字幕位图。

### 需要保留的能力

原生字幕窗口必须继续支持现有配置：

- 字幕宽度百分比；
- 距屏幕底部百分比；
- 字号；
- 最大行数；
- 字幕背景透明度；
- 淡入时间；
- 停留时间；
- 淡出时间。

还必须保留以下行为：

- 新翻译结果覆盖旧结果；
- 迟到的旧请求不能覆盖新字幕；
- 翻译前后文本相同时不显示字幕；
- 划词翻译成功后必须显示结果；
- 字幕显示期间不抢焦点、不阻塞鼠标；
- 应用退出时正确销毁原生窗口和渲染资源。

### 应删除的旧字幕逻辑

原生字幕窗口完成后，不应保留两套字幕实现。建议删除：

- ` main.go:38 ` 的 Wails 字幕窗口创建；
- ` main.go:76 ` 的 Wails 字幕窗口参数及本次失败的 Composition 配置；
- ` main_windows_test.go:11 ` 只验证 Wails 参数的测试；
- ` frontend/src/App.vue:7 ` 开始的字幕事件、状态和计时逻辑；
- ` frontend/src/App.vue:124 ` 的字幕模板；
- ` frontend/src/style.css:45 ` 开始的 WebView 字幕样式；
- 与 Vue 字幕展示相关、已经不再覆盖真实渲染链路的前端测试。

设置页面及其 Wails 运行时逻辑继续保留。

## 建议实施顺序

1. 定义与平台无关的字幕控制器接口；
2. 将 ` app.go:493 ` 的布局计算结果传给字幕控制器；
3. 将 ` app.go:525 ` 的翻译结果直接交给字幕控制器，不再发送 Wails 前端事件；
4. 实现 Windows 原生分层窗口、文本布局和逐像素透明渲染；
5. 实现淡入、停留和淡出状态机；
6. 将字幕窗口生命周期接入应用启动、配置刷新和退出流程；
7. 删除 Wails/Vue 字幕窗口及相关旧代码；
8. 补充单元测试、Windows 集成测试和人工截图验收。

## 验收标准

改造完成后至少验证以下场景：

1. 字幕以外区域能完整看到底层应用，不存在白色或黑色矩形；
2. 不存在标题栏、边框、关闭按钮和任务栏入口；
3. 鼠标点击字幕所在区域时，底层应用仍能正常响应；
4. 字幕显示时不会切换当前焦点；
5. 中英文、多行、长文本和最大行数限制显示正确；
6. 宽度和距底部百分比修改后立即生效；
7. 淡入、停留和淡出时间符合配置；
8. 连续快速翻译时只保留最新结果；
9. 剪切板翻译和划词翻译都能显示字幕；
10. 退出应用后没有残留窗口、托盘图标或后台进程。
